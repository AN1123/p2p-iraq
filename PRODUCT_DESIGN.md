### تدفق المشتري (شراء USDT بـ IQD)

```
[الموقع] المستخدم يختار "شراء USDT"
    ↓
[نموذج] يُدخل:
  - المبلغ بالدولار أو بالدينار
  - شبكة USDT (TRC20 / ERC20 / BEP20)
  - عنوان محفظته
  - طريقة دفعه (FIB / ZainCash / Qi Card)
  - محافظته
  - رقم تيليجرامه
    ↓
[الموقع] يحسب:
  - المبلغ بالدينار = USDT × سعر البيع
  - العمولة 1% = تُضاف للمبلغ (يدفعها المشتري)
  - رسوم الشبكة تُعرض كتحذير
    ↓
[الموقع] يولّد:
  - Order ID (مثال: BUY-20260303-0042)
  - رسالة جاهزة للنسخ
  - Deep Link لفتح البوت مباشرة
    ↓
[زر واحد] "افتح تيليجرام وأرسل الطلب"
  → يفتح البوت مع start parameter يحتوي Order ID
    ↓
[البوت] يستلم الطلب → يرسل تأكيد + رقم حساب الدفع
    ↓
[المستخدم] يدفع عبر FIB/ZainCash/Qi Card
    ↓
[المستخدم] يرفع لقطة الدفع للبوت
    ↓
[المشرف] يتحقق → يطلق USDT من escrow → يرسله للمحفظة
    ↓
[البوت] يُرسل تأكيد الإتمام للطرفين
```

### تدفق البائع (بيع USDT مقابل IQD)

```
[الموقع] المستخدم يختار "بيع USDT"
    ↓
[نموذج] يُدخل:
  - مقدار USDT المراد بيعه
  - الشبكة
  - طريقة الاستلام (FIB / ZainCash / Qi Card)
  - رقم الحساب المحلي لاستلام IQD
  - محافظته + تيليجرام
    ↓
[الموقع] يحسب:
  - سيستلم بالدينار = USDT × سعر الشراء
  - العمولة 1% = تُخصم من المبلغ (يدفعها البائع)
    ↓
[Deep Link → البوت]
    ↓
[البوت] يعطيه عنوان محفظة الـ Escrow
    ↓
[البائع] يرسل USDT للـ Escrow
    ↓
[البوت/المشرف] يتحقق من وصول USDT on-chain
    ↓
[المشرف] يحوّل IQD للبائع عبر طريقة الدفع المختارة
    ↓
[البائع] يؤكد الاستلام
    ↓
[البوت] يُتمّ الصفقة ويُطلق التأكيد
```

### مطابقة الطلبات: الخيار الأفضل للـ MVP

**التوصية: خيار A (يدوي) مع بوت مساعد**

**السبب:**
- العراق سوق يعتمد على الثقة الشخصية
- المطابقة التلقائية تحتاج سيولة كافية مسبقاً
- الأخطاء في المطابقة التلقائية = نزاعات = ضرر للسمعة
- يمكن الترقية لـ خيار B بعد 3 أشهر وحجم تداول ثابت

**آلية العمل اليدوي:**
```
1. طلب جديد يصل لمجموعة المشرفين (Telegram Group)
2. المشرف المناوب يأخذ الطلب خلال 5 دقائق
3. يطابقه مع عروض التجار الموثوقين (قائمة Google Sheet / DB)
4. يخطر الطرفين ويبدأ الـ Escrow
```

### احتيال شائع في العراق وطرق مكافحته

| نوع الاحتيال | الوصف | الحل |
|-------------|-------|------|
| **لقطة مزوّرة** | يرسل صورة دفع معدّلة | التحقق مباشرة من رقم العملية مع البنك/ZainCash |
| **Chargeback FIB** | يعكس التحويل بعد استلام USDT | لا تُطلق USDT إلا بعد confirmation مؤكد (ليس pending) |
| **Double spend** | يرسل لعنوانين بنفس المبلغ | تأكيد on-chain بـ 3+ confirmations للـ TRC20 |
| **Social engineering** | ينتحل شخصية المشرف | تنبيه ثابت: المشرف لن يطلب مالاً خارج المنصة |
| **Account takeover** | يستولي على حساب تيليجرام | 2FA إجباري للمشرفين + تنبيه للمستخدم عند تغيير البيانات |
| **Fake escrow** | ينشئ مجموعة مشابهة | رقم بوت رسمي مثبّت + توثيق القناة |

## 4. Escrow واقعي

### المستويات الثلاثة

**L1: بدون Escrow (وسيط + تحقق يدوي)**
```
المخاطر:
- الطرف الأول يدفع، الثاني يختفي
- لا ضمان للطرفين
- يعتمد 100% على ثقة الوسيط
- المناسب فقط للمستخدمين الـ VIP المعروفين شخصياً
```
⛔ غير مناسب للعموم

---

**L2: Escrow يدوي (OTC-Escrow) ← التوصية للإطلاق**
```
آلية البيع:
1. البائع يرسل USDT لمحفظة الـ Escrow الخاصة بك
2. أنت تتحقق on-chain (3 confirmations)
3. تأكيد للمشتري بالدفع
4. المشتري يدفع IQD لحسابك المحلي
5. أنت تتحقق من الاستلام
6. تُطلق USDT للمشتري من محفظة الـ Escrow

المزايا:
✅ بسيط وقابل للتنفيذ فوراً
✅ كل صفقة تمر عبرك = تحكم كامل
✅ مناسب للحجم الصغير

المخاطر:
⚠️ أنت تحمل USDT مؤقتاً = مسؤولية
⚠️ إذا اختُرقت محفظتك = خسارة
⚠️ يحتاج سيولة دائمة

الحل: محافظ منفصلة لكل شبكة + Cold wallet للاحتياطي
```
✅ **الأفضل للـ MVP**

---

**L3: Escrow آلي على السلسلة (Smart Contract)**
```
آلية العمل:
- Smart contract على TRON/BSC يحتجز USDT
- يُطلق تلقائياً بشرط معين (Multi-sig أو Oracle)
- لا يحتاج ثقة بالوسيط لإطلاق الأموال

التعقيد:
❌ يحتاج تطوير smart contract = 3-6 أشهر
❌ تكاليف audit أمني عالية
❌ المستخدم العراقي يحتاج تعليم على المحافظ
❌ رسوم gas قد تؤثر على صغار المعاملات
```
⏳ خطة مستقبلية بعد نمو المنصة

---

### كيف تشرح للمستخدم بسهولة؟

> **"محفظة الضمان"** - نحن نحتجز الدولار بشكل مؤقت وآمن حتى يتم التأكد من دفع الطرفين. فقط بعد التأكيد يُطلق المبلغ. هذا يحمي الجميع.

## 5. التسعير والرسوم

### هيكل الأسعار

```
سعر المرجع (مثلاً: Binance P2P / سعر السوق المحلي)
    + هامش المنصة (Spread): 0.5% – 1%
    + عمولة 1% على المعاملة

سعر الشراء (تشتري USDT منا):  سعر مرجع + spread
سعر البيع (تبيع USDT لنا):    سعر مرجع - spread
```

---

## 6. تصميم بوت تيليجرام

### الأوامر الأساسية

```
/start     - رحبة + قائمة رئيسية
/buy       - بدء طلب شراء
/sell      - بدء طلب بيع
/track     - متابعة طلب برقمه
/rates     - أسعار اليوم
/proof     - رفع إثبات الدفع
/dispute   - رفع نزاع
/cancel    - إلغاء الطلب الحالي
/help      - دليل الاستخدام
/admin     - لوحة المشرف (مشرفين فقط)
```

---

### State Machine للصفقة

```
IDLE
  ↓ /buy أو /sell
COLLECTING_INFO
  ↓ (أسئلة متسلسلة: مبلغ → شبكة → محفظة → دفع → محافظة)
ORDER_CREATED
  ↓ (إرسال تعليمات الدفع)
WAITING_PAYMENT
  ↓ (المستخدم يرفع لقطة)
PROOF_SUBMITTED
  ↓ (المشرف يراجع)
  ├── PROOF_REJECTED → طلب دليل أفضل → WAITING_PAYMENT
  └── PROOF_APPROVED
        ↓
      PROCESSING (إطلاق USDT / تحويل IQD)
        ↓
      COMPLETED ✅
      
في أي مرحلة:
  /cancel → CANCELLED
  مشبوه → FROZEN ❄️
```

## 8. خطة التشغيل على مستوى العراق

### هيكل المشرفين

| الدور | العدد المقترح | المسؤولية |
|-------|--------------|-----------|
| مشرف مناوبة | 2-3 | معالجة الطلبات العادية |
| مشرف أول | 1 | النزاعات + حالات التجميد |
| مسؤول المالية | 1 | إدارة السيولة + التسوية |
| مدير عام | 1 (أنت) | قرارات العليا + VIP |

**جدول المناوبات (توصية):**
```
الورديه الصبحية:  08:00 – 16:00 (مشرفان)
الوردية المسائية: 16:00 – 24:00 (مشرفان)  ← أنشط فترة
الوردية الليلية:  24:00 – 08:00 (مشرف واحد on-call)
```

---

### سياسة حل النزاعات

| المرحلة | الإجراء | SLA |
|---------|---------|-----|
| 1 | طرفان يتواصلان بإشراف المشرف | 1 ساعة |
| 2 | مشرف أول يراجع كل الأدلة | 4 ساعات |
| 3 | قرار نهائي + تعويض إن وجد | 24 ساعة |
| 4 | رفض + حظر + إشعار قانوني (احتيال) | فوري |

**متى نرجع الأموال:**
- ✅ دفع ثابت بالرقم الرسمي لكن USDT لم يُرسَل
- ✅ خطأ تقني من جانبنا
- ✅ فشل in-chain لا يد للمستخدم فيه

**متى لا نرجع الأموال:**
- ❌ المستخدم أرسل لعنوان خاطئ
- ❌ إثبات مزوّر
- ❌ انتهاء مدة الطلب بدون دفع

---

### إدارة السيولة

**المشكلة:** P2P لكن أنت الوسيط، فتحتاج USDT دائماً متاحاً للـ Escrow.

**الحل على 3 مستويات:**

```
المستوى 1 (الإطلاق): احتفظ بـ 2,000-5,000 USDT في محفظة Escrow
المستوى 2 (نمو): شبكة تجار موثوقين يضعون عروض مسبقاً
المستوى 3 (تحجيم): Offer Book تلقائي + مطابقة ذكية
```

**لضمان وجود سيولة:**
- تحديد وقت استجابة أقصى (5 دقائق للمطابقة)
- نظام انتظار smart: إذا لا يوجد سيولة → وضع الطلب في Queue + إشعار المستخدم
- علاوة سيولة للتجار في المحافظات النائية
